Structure

Main Parts

Basic HTML code contains the following structural elements:

Element Tags Purpose
Document type declaration <!DOCTYPE html> Indicates to the browser that the document type is HTML
HTML <html></html> Indicates the start and end of the HTML document
Head <head></head> Contains metadata "data about data"; document properties about the HTML document
Title <title></title> Defines the title of the webpage on the browser tab and search results
Body <body></body> Contains all visible elements of the webpage
Syntax: 
<!DOCTYPE html>
<html>
<head>
  <title>HTML Basics</title>
</head>
<body>

  <h1>Structure</h1>
  <p>This page talks about the basic structure of HTML.</p>

</body>
</html>

Other Parts

Part Tags Purpose Semantic?
Main <main></main> contains primary content unique to the webpage Yes
Section <section></section> divides content thematically Yes
Article <article></article> contains stand-alone content that can make sense outside the current page Yes
Aside <aside></aside> contains content indirectly, if at all, related to the main content Yes
Header <header></header> contains introductory elements for the parent or repeating introductory content across webpages Yes
Footer <footer></footer> contains secondary information for the parent or repeating secondary content across webpages Yes
Division <div></div> all purpose structural element, demarcates a block part of the code No
Span <span></span> demarcates an inline element that DOES NOT occupy an entire line part of the code No

Nesting

An important aspect of HTML is nesting. This refers to the hierarchical organization of elements. Some elements are broader and contain elements within. The former is called the parent, while the latter are called children (which can also be parents if containing sub-elements). Children elements within the same parent element are called siblings.

Syntax: 
<body>
  <main>
    <section>
      <p>Paragraph</p>
      <ul>
          <li>List</li>
      </ul>
    </section>
    <section>
      <p>Paragraph</p>
      <ul>
          <li>List</li>
      </ul>
    </section>
  </main>
<body>

Block and Inline Elements

Block and inline assigments determine the nature of the elements and how they interact with the lines of the webpage.

Element

The element is the building block of HTML. It defines the content to be presented by the browser. program or application that displays webpages It is comprised of characters letters, numbers, symbols, and spaces arranged according to a particular syntax. rules governing how a language is structured

Examples of characters:

  • a is a character
  • 1 is a character
  • ! is a character
  • (space) is a character

Parts of an Element

An element has specific parts that must be present to be understood and executed by the browser. It is initiated by the start tag, aka "open" tag contains element content, the actual content to be presented by the browser and ends with the end tag aka "close" tag .

  1. Start Tag
    1. Opening angle bracketaka the "less than symbol" <
    2. Tag nameelement identifier x
    3. Closing angle bracketaka the "greater than symbol" >
  2. Element Content abc
  3. End Tag
    1. Opening angle bracket <
    2. Forward slashindicates closing tag /
    3. Tag name x
    4. Closing angle bracket >

Some elements are empty which means that they do not have element content nor end tags (Example 2).

Start Tag Element Content End Tag
Regular Element <x> abc </x>
Empty Element <x>

Attribute

An attribute is an aspect of an element. It provides additional information that serves to further specify instructions for the browser. They can be applied to all elements, but most do not require them, save for some. e.g. <a>, <img>, <iframe>, <input> To be applied, the attribute name of the attribute to be defined and corresponding value, how the browser presents the element content based on chosen attribute if any, must be encoded inside the start tag.
