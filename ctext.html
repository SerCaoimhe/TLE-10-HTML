Text

Text is an important part of any webpage. As discussed in the Text chapter of HTML, it can come in the form of letters, numbers, symbols, and even a space character.

Font

As text can be handwritten in a variety of ways, so can it be rendered in the digital space. This aspect of text is what is referred to as font. For example, in a word processor like Microsoft Word, fonts can vary from Arial, to Calibri, to 'Times New Roman', among others.

Font Properties
Property Syntax Description
Style font-style: sets style to normal, italic, or oblique
Variant font-variant: determines whether small capital characters (small caps) will be applied
Weight font-weight: defines text thickness
Stretch font-stretch: defines text wideness
Size font-size: defines text size
Family font-family: defines the font family to be applied
Font font: shorthand property with values defined in the order following the order of the properties listed above (style, variant, weight, etc.)

Style

Style sets the uprightness or slantness of text. Unless defined, the font will be rendered in its default state. This is usually upright (not slanted), but there are exceptions like Photograph Signature which already appear italicized by default.

Italic and oblique styles are confusing because they seem to behave similarly. However, there is a cosmetic and semantic difference. Italic is a deliberate, slanted version of the font altogether. Sometimes its letter design is different from the normal version's. On the other hand, oblique simply slants text by force. It uses the same characters but angled.

Use italic for slanted text unless a synthetic or artificial slant is intended, in which case, use oblique. If a font does not have italics, the browser will attempt to apply oblique anyway.

Variant

Use variant when small capitals uppercase letters with lowercase height (small caps) are desired. Small caps are used for subtler emphasis compared to all caps, Latin phrases, and acronyms (e.g. pagasa, am, pm), among others.

Weight

Weight sets the thickness of text. Values include normal, lighter, bold, and bolder, among others. Thickness can also be set using integer values from 100 to 1000.

Compatible values will vary from font to font. Some fonts do not have bolder weight, for example; some do not have 100; so on and so forth.

Stretch

Stretch sets the width of the characters and, in effect, the text overall. Values include condensed, normal, and expanded, among others. Stretch can also be set using percentage values.

Compatible values will vary from font to font. Some fonts do not have condensed stretch, for example; some do not have 50%; so on and so forth.

Family

Family sets the design of the text. A font family contains different iterations of a font in terms of style, weight, and stretch. This is the reason why possible property values for each font varies; some font families are more comprehensive than others.

When discussing font family, it is also helpful to understand the concept of font fallbacks. When setting a font family, it is recommended that font fallbacks are also specified. Fallbacks are backups in case the originally set family is unable to be rendered for whatever reason. The browser will try to render each one in order until it succeeds.

Examples of the recommended syntax for the font-family property is as follows:

Font Family and Fallbacks
Property Value
Named Fonts Generic Fonts
font-family: Arial, Calibri, sans-serif
font-family: "Roboto Flex", Montserrat, sans-serif
font-family: "Courier New", Courier, monospace
Font names comprised of more than one word must be enclosed in quotation marks as seen above.

There are five generic font families which can be used as the last resort font family values:

  • Serif â€“ have ornate character ends and corners (e.g. Times New Roman)
  • Sans-serif â€“ have simple character ends and corners (e.g. Calibri)
  • Monospace â€“ have equally-wide characters (e.g. Courier New)
  • Cursive â€“ have handwritten-styled and connected characters (e.g. Photograph Signature)
  • Fantasy â€“ have decorative or medieval-styled characters (e.g. Papyrus)

Including fallbacks ensures that text will be rendered as closely as intended in whatever situation.
