What is a form?

A form is a data gathering instrument containing various input fields aka "form fields" that respondents or data collectors fill out with the appropriate information. In the context of HTML, input fields are referred to as form controls. Websites can make use of forms for various purposes such as collecting registration details, login credentials, or feedback.

Elements

The form element is applied using the form <form> tag, filled with the form control (input fields, as previously mentioned) and auxilliary elements. Essentially, the form element merely contains the form controls and support elements, without which it is useless.

The following is a list of common form elements:

Form and Form Controls
Element Tags Attributes Purpose
Form <form></form> action, target, method, oninput* Creates the form
Input <input> id, name, value, placeholder, required, disabled, readonly, size type Creates a field for several types of responses
Text Area <textarea></textarea> id, name, value, placeholder, required, disabled, readonly rows, cols Creates a multiline field for text responses
Select <select></select> id, name, required, disabled, size multiple Creates a drop-down field for predetermined responses

Auxilliary
Element Tags Attributes Purpose
Label <label></label> for Defines the form control label
Field Set <fieldset></fieldset> Creates a group for related form controls
Legend <legend></legend> Defines the field set caption
Data List <datalist></datalist> id Creates a predetermined list of responses for an input field
Option Group <optgroup></optgroup> label Creates groups for related options in a drop-down list
Option <option></option> value Defines an option in a drop-down list
Button <button></button> type, onclick* Creates a clickable button
Output <output></output> id, name, for Displays the result of a calculation

*oninput and onclick attributes can be replaced by other event attributes attributes (e.g. onchange, onmouseover, onkeydown) that indicate the trigger for a JS action

Attributes

Before proceeding to the other aspects of forms, it is essential to be familiar with common attributes as these are essential in understanding and applying proper syntax.

Form and Form Controls
Attribute Element Purpose
id input, text area, select defines unique identifying code for styling, links form control to its label
name input, text area, select defines label attached to the submitted response, often abbreviated
value input, text area, option defines value of the default or submitted response
placeholder input (text types), text area defines displayed sample response
required input, text area, select sets form control as required for submission
disabled input, text area, select sets form control as uneditable and excluded from submission
readonly input (text types), text area sets form control as uneditable but included in submission

Examples

Preview Syntax
<input type="text" id="ex1" name="ex1" placeholder="placeholder (editable, excluded from submission unless edited)" required size="50">
<input type="text" id="ex2" name="ex2" value="disabled (uneditable, excluded from submission)" disabled size="50">
<input type="text" id="name2" name="ex3" value="value (editable, included in submission)" size="50">
<input type="text" id="ex4" name="ex4" value="value + readonly (uneditable, included in submission)" readonly size="50">

Label

The label element, as the name suggests, provides a label to a form control in order to clarify what information is being gathered. It is applied using the label <label> tag and the for attribute with the syntax <label for="x">y</label>, where x is the id value of the corresponding form control and y is the actual label.

Attribute Value Element Content
<label for= "x" > y </label>

The label's for attribute value should match the form control's id attribute value.

<label for="x">
<input id="x">

This ensures that the label corresponds to the form control. When the label is clicked, the input field will be activated.

For radio or checkbox input, click the actual option label.



Syntax: 
<form action="action_page.php" method="POST"> 
  <div> 
    <label for="gname">Given Name:</label> 
    <input type="text" id="gname" name="gname" placeholder="Caoimhe Saoirse" required> 
  </div>
  <div>
    <label for="age3">Age:</label>
    <input type="number" id="age3" name="age" placeholder="18" required>
  </div>
</form>

Field Set and Legend

The field set element groups related form controls for better organization. This is not required but is recommended for longer or more complicated forms. It is applied using the <fieldset> tag.

Often placed inside the fieldset element is the legend element which adds a name to the field set. It is applied using the <legend> tag.

Syntax: 
<form action="action_page.php" method="POST"> 
  
<fieldset>
    <legend>Identification</legend>
    <div> 
      <label for="gname2">Given Name:</label> 
      <input type="text" id="gname2" name="gname" placeholder="Adam Antium"> 
    </div> 
    <div> 
      <label for="sname">Surname:</label> 
      <input type="text" id="sname" name="sname" placeholder="Grrrah"> 
    </div> 
</fieldset>

  <br>
  
<fieldset>
    <legend>Contact Details</legend> 
    <div> 
      <label for="email">Email:</label> 
      <input type="email" id="email" name="email" maxlength="89" placeholder="email@domain.com"> 
    </div>
    <div> 
      <label for="phone">Phone No.:</label> 
      <input type="tel" id="phone" name="phone" placeholder="(012)-345-6789" required> 
    </div>
</fieldset>

</form>

Form Controls

Form controls are the elements that provide input fields where respondents provide data. These come in three main forms: input, text area, and select.

Input

The input element creates several kinds of form controls (aka input types). It is applied using the <input> tag and defined with the type attribute.

The following are the possible input types, each with its own appropriate use:

Focus in this class
Type Attribute Value Description
Checkbox checkbox Sets input type to a checkbox, allowing respondent to select multiple answers from a set of checkboxes toggleable boxes that are either marked (selected) or unmarked (not selected)
Date date Sets input type to a date, prompting respondent to choose from a pop-up calendar
Email email Sets input type to email, prompting respondent to enter a valid email
File file Sets input type to file upload, prompting respondent to select a file from the device to upload
Number number Sets input type to a number, prompting respondent to enter a number
Password password Sets input type to text that is masked (e.g. asterisk)
Radio radio Sets input type to a radio button, allowing respondent to select only one answer from a set of radio buttons toggleable circular buttons that are either marked (selected) or unmarked (not selected)
Range range Sets input type to a numerical slider, prompting respondent to move the slider to choose a number within the defined range
Telephone tel Sets input type to a telephone number, prompting respondent to enter a valid telephone number
Text text Sets input type to text, prompting respondent to enter a text response
Checkbox

Each checkbox is its own input element with its own label. Each one is connected to the main label (e.g. question being asked) through the name attribute.

The name value should match for all checkboxes and the for value of the question label. Additionally, each label-input pair should have matching for and id values.

Preview Syntax 




<form>
  <label for="q">Liked terminal punctuation</label>
  <br>
  <input type="checkbox" id="?" name="q" value="happy">
  <label for="?">?</label>
  <br>
  <input type="checkbox" id="!" name="q" value="sad">
  <label for="!">!</label>
  <br>
  <input type="checkbox" id="." name="q" value="hangry">
  <label for=".">...</label>
</form>
Radio

Each radio button is its own input element with its own label. Each one is connected to the main label (e.g. question being asked) through the name attribute. Unlike the checkbox, only one radio button can be selected at a time by default.

The name value should match for all radio buttons and the for value of the question label. Additionally, each label-input pair should have matching for and id values.

Preview Syntax 



<form>
  <label for="coolness">Are you an aura farmer?</label>
  <br>
  <input type="radio" id="happy" name="coolness" value="happy">
  <label for="happy">Always</label>
  <br>
  <input type="radio" id="sad" name="coolness" value="sad">
  <label for="sad">Sometimes</label>
  <br>
  <input type="radio" id="hangry" name="coolness" value="hangry">
  <label for="hangry">Never</label>
</form>
