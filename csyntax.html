Syntax

If the basic building block of html is the element, in CSS, it is the rule. The rule itself can be split into two main segments, the selector and the declaration block, which in turn is composed of at least one declaration.

A declaration is composed of the property and value pair. Multiple property-value pairs are separated by semicolons.

Parts of the CSS rule:
  1. Selector x
  2. Declaration Block
    1. Open curly bracket {
    2. Declaration
      1. Property y
      2. Colon :
      3. Value z
      4. Semicolon ;
    3. Closing curly bracket }
Selector Declaration Block
x {y1: z1; y2: z2;}

The syntax in the table above applies when the rule is put in a single line. However, for more detailed rules, it is recommended to split it into several lines for clarity.

Multiline rule syntax: 

        x {
        y1: z1;
        y2: z2;
        y3: z3a z3b;
        }

Properties can have multiple values as shown in the third property, y3.

Selectors

The selector is an essential part of CSS. It is the part of the rule that determines which element or elements are affected. There are several types of selectors, making styling highly flexible and precise.

Individual Selectors

Selector Syntax Purpose
Universal * {declaration block} selects all elements
Element element {declaration block} selects the defined element
Class .class {declaration block} selects elements with the defined class attribute value
Element-Class element.class {declaration block} selects only the defined element with the defined class attribute value
ID #id {declaration block} selects the elements with the defined id attribute value

Element Selector

Syntax: 

 h3  {
   font-style: italic;
 }
 p  {
   text-transform: uppercase;
 }
 table  {
   background-color: aquamarine;
 }

Syntax:

  color: yellow;

Group Selectors

Multiple selectors may be used in one rule to apply the same style to all of them. This makes the code less cluttered and more efficient. To use group selectors, simply separate the selectors with commas.

Example 1: multiple element selectors

Syntax: 

h3, h4 {
  color: rgb(83, 186, 198);
}
p, table {
  background-color: rgb(114, 250, 114);
}
Preview:
Example 2: mixed selectors

Syntax: 

.und, #und1 {
  text-decoration: underline;
}
h4, .strike {
  text-decoration: line-through;
  background-color:rgb(254, 227, 91);
}

Combinators

Types

There are three ways to add styles to an HTML document. In other words, there are three types of CSS-to-HTML integration. These three are not mutually exclusive, meaning they can exist simultaneously. However, each type is given a specific priority when it comes to implementation.

The hierarchy is as follows from highest priority to lowest:

  1. Inline
  2. Internal
  3. External

The browser will always implement higher priority CSS when available. It will override properties applied using lower priority CSS.

Inline

Inline CSS refers to CSS encoded directly into html elements. This is whenever the style attribute is applied.

Syntax: 
<div style="background-color: salmon; color:white">Inline CSS ftw</div>
Preview:
Inline style ftw

Internal

Internal CSS refers to CSS encoded in the style element within the head element of the HTML document.

This is recommended for localized styles that vary from page to page.

Syntax: 

<head>
  <style>
    div {
      background-color: rgb(115, 202, 159);
      color:white;
    }
  </style>
</head>
<body>
  <div>Internal CSS ftw</div>
</body>
Preview:
Internal CSS ftw

External

External CSS refers to CSS encoded in a separate .css document and linked using the <link rel="stylesheet" href="filename.css"> element to the HTML document in the head element of the HTML document.

This is recommended for big projects where consistency and convenience of editing is crucial as all styles is found in a single document.

Syntax: 
<head>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <div id="externalexample">External CSS ftw</div>
</body>

Specificity

Specificity refers to the priority given to rules based on the selectors used. There are cases when rules overlap because selectors are used multiple times in a stylesheet or internally. In such cases, the browser will implement the rule with the highest specificity.

The selectors discussed in the previous section are presented in order of decreasing specificity:

  1. ID
  2. Element-Class
  3. Element
  4. Universal

In addition to the selector, specificity also considers the CSS type. The types discussed in the previous section are presented in order of decreasing specificity:

  1. Inline
  2. Internal
  3. External

In summary, specificity dictates that inline styles will be implemented before any other. This will be followed by internal styles. The internal styles, in turn, will be checked for specificity based on selectors. Finally, external styles will be implemented based on selectors.

The following is the specificity hierarchy in order of decreasing specificity:

  1. Inline CSS
  2. Internal CSS
    1. ID
    2. Element-Class
    3. Class
    4. Element
    5. Universal
  3. External CSS
    1. ID
    2. Element-Class
    3. Class
    4. Element
    5. Universal

The browser will implement styles following the list above.

Example:

Syntax: 

.und, #und1 {
  text-decoration: underline;
}
h4, .strike {
  text-decoration: line-through;
  background-color:rgb(254, 227, 91);
}
